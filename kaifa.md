# 🧠 AI驱动人生模拟器（SillyTavern风格）开发需求文档

## 📌 1. 项目概述

本项目是一个“AI驱动的轻量人生模拟器系统”，融合：
- 状态模拟（体力 / 心情 / 金钱）
- AI角色对话系统
- 世界书规则系统（Worldbook）
- 记忆系统（Memory）
- 事件驱动叙事系统

目标是构建一个类似 **SillyTavern + 人生模拟器 + 规则引擎** 的可扩展AI叙事框架。

---

## 🎯 2. 核心目标

系统需要实现以下能力：

1. 玩家拥有基础状态系统（体力 / 心情 / 金钱）
2. 每天自动推进“时间循环（Day Tick）”
3. AI根据状态生成事件与剧情
4. 用户可进行选择影响世界状态
5. 系统支持世界书规则自动触发
6. 支持长期记忆记录与调用

---

## 🧩 3. 系统模块设计

### 3.1 状态系统（Core Stats）

玩家状态结构：

```json
{
  "energy": 100,
  "mood": 70,
  "money": 500
}
```

### 3.2 AI对话与叙事系统（Narrative Engine）

功能：

- AI负责生成剧情描述
- AI生成事件选项
- AI根据状态调整语气与内容

输入：

- 当前状态
- 当前记忆
- 当前世界规则

输出：

- 剧情文本
- 可选行动列表

------

### 3.3 世界书系统（Worldbook）

用于定义“规则触发逻辑”。

支持：

- 关键词触发
- 状态触发
- 时间触发

示例：

```
{
  "trigger": "energy < 30",
  "effect": "触发疲劳事件，降低工作效率",
  "priority": 10
}
```

------

### 3.4 记忆系统（Memory System）

用于长期存储玩家行为与世界变化。

结构：

```
{
  "events": [
    {
      "day": 1,
      "content": "玩家失去了100元",
      "impact": "money -100"
    }
  ],
  "relationships": {},
  "flags": {}
}
```

用途：

- AI上下文注入
- 长期剧情发展
- NPC关系变化

------

### 3.5 事件系统（Event Engine）

每日自动生成事件：

类型包括：

- 工作事件
- 随机事件
- 健康事件
- 社交事件
- 金钱事件

流程：

1. 状态输入AI
2. AI生成事件
3. 玩家选择
4. 更新状态 + 记忆

------

### 3.6 Prompt系统（角色与世界设定）

支持：

- 角色卡（Character Card）
- 世界设定（World Prompt）
- 动态变量注入

变量示例：

```
{{energy}} {{mood}} {{money}}
```

------

## 🔁 4. 核心运行循环（Game Loop）

系统每天执行一次：

```
1. 更新玩家状态
2. 检查世界书规则
3. AI生成事件
4. 玩家做出选择
5. 更新状态
6. 写入记忆
7. 输出下一天剧情
```

------

## 🧠 5. AI系统架构

建议采用双AI结构：

### Planner（规划AI）

- 生成事件结构
- 拆解剧情逻辑
- 输出JSON任务

### Executor（执行AI）

- 根据事件生成文本
- 输出UI内容
- 生成可选行动

第一版本必须实现：

- 状态系统
- 每日事件生成
- 玩家自定义
- 状态变化（人体生理需求如睡眠欲，饥渴欲，性欲、生理期等贴近正常人体的因素）
- 简单记忆记录
